Case Study - myFlix App (React)

Project on GitHub
Project Live
Quick Overview

Overview

The myFlix application was designed as a movie web app, utilizing the MERN stack to create both a server-side REST API and a client-side Single Page Application (SPA). The goal of the project was to provide users with seamless access information about movies, manage personal profiles, and curate their own list of favorite movies, all within a responsive, intuitive user interface.

Purpose & Context

myFlix is a personal project I created to demonstrate my abilities in full-stack JavaScript development.

Objectives

The primary challenge was to create a fully functional, user-friendly web application that would interact with a server-side API to manage movie data and user accounts. The key objectives were:

  • Develop a RESTful API to handle data related to movies, genres, and directors.
  • Build a responsive and dynamic front-end interface using React.
  • Implement user authentication and authorization.
  • Ensure the application was deployed and accessible online.

Process

1. Server-Side Development

Database Design: Using MongoDB, I defined the data models for movies, genres, directors, and users. Mongoose was employed to create schemas and enforce data validation.

API Development: I developed a RESTful API using Node.js and Express. This API included endpoints for retrieving movie information, managing user accounts, and handling user authentication. Key features included:

  • User Management: Endpoints for user registration, login, updating profiles, and deregistration.
  • Movie Management: Endpoints for retrieving movie details, genres, and director information.
  • Favorites Management: Endpoints for users to add and remove movies from their favorite list.

Authentication: Implemented JWT-based authentication for securing API endpoints. User passwords were hashed using Bcrypt to ensure security.

Deployment: The API was deployed on Heroku, with the database hosted on MongoDB Atlas. I configured environmental variables to manage sensitive data securely.

2. Client-Side Development

UI Design and Prototyping: I started with wireframes and prototypes to design the user interface. The focus was on creating a clean, intuitive layout that would be easy to navigate.

React Development: The SPA was built using React, taking advantage of its component-based architecture to ensure modularity and reusability. React Router was used to manage client-side routing, enabling users to navigate between the login/signup views, the main movie list, and the profile page.

State Management: I used React's state and props to manage the application's state, ensuring that data was efficiently passed between components.

Responsive Design: The application was styled using React Bootstrap and custom SCSS to ensure it was fully responsive across different devices. The design was tested and refined to provide a consistent user experience on both desktop and mobile devices.

Integration with API: The client-side was integrated with the server-side API to fetch movie data, manage user accounts, and handle user authentication.

3. Testing and Debugging

Unit Testing: Individual components and functions were tested to ensure they behaved as expected. The API was tested using Postman, the client-side components were tested using mainly console logs, the IDE debugger, and individual interaction in the app.

Integration Testing: The interaction between the client-side and server-side components was tested to ensure smooth data flow and proper API responses.

User Testing: I asked several users to test the application to gather feedback on the its usability and performance. Based on this feedback, several UI elements were refined, and minor bugs were fixed.

4. Deployment

The final step was to deploy the client-side application on Netlify, making it accessible to users. I configured continuous deployment to automatically update the application whenever new changes were pushed to the repository.

Challenges and Solutions

Handling Asynchronous Data Fetching

One of the challenges I faced was managing asynchronous data fetching, especially when multiple API calls were required to render a single view. To address this, I used React's useEffect hook to fetch data after the component mounted and employed conditional rendering to ensure the UI only displayed once all data was available.

Authentication and Security

Implementing secure user authentication was critical. I used JWT tokens for securing API endpoints and ensured sensitive data, like user passwords, was properly hashed. Additionally, CORS was implemented to prevent unauthorized access from other domains.

Optimizing Performance

To improve the application's performance, I optimized the React components and reduced the number of unnecessary re-renders.

Retrospective

What went well?

The myFlix project was a comprehensive learning experience that allowed me to gain deep insights into full-stack development using the MERN stack. I learned how to create and set up a back-end and how to built a non-relational database which was all completely novel to me. With React, I discovered an exciting world of advanced JavaScript UI libraries and component-based architecture and this approach came across as very logical, neat, and efficient.

What didn't go well?

My biggest issues were stemming from managing my learning curve, project requirements, and time. If I had to start over, I'd give myself more time at the beginning to play around with different concepts and breaking the solution down into even smaller chunks which likely would have cost me more time initially but streamline my development process further down the road with better organisation and understanding.

What are the next steps?

The app deserves some additional features such as information about actors, filtering/displaying featured movies, movie ratings and commentary, showing similar movies, movie trailers, and 3rd-party login such as Google OAuth.
